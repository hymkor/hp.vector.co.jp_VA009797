Vio File Manager‘eff’FAQ Page

バージョンアップしたら、ディレクトリを開くのが遅くなったのですが…
設定ファイル _eff を上書きしていませんか？_eff に記述されている print 文で、+l +c が記述されていると、ディレクトリ移動の度に、拡張属性を読み込むので、遅くなります。特にファイルにコメントなどを記述するつもりが無いのでしたらprint文に -l -c と記述してください。
eff のキーバインドは原形が謎ですが、いったい、何をモデルに…
基本的に vi , mnews , Emacs , rogue(ボカッ)といったところです。基本的に以下のように、右手だけで操作できるようになっています。
I(入)O(出)
H(←) J(↓)K(↑)L(→)
I は、ページャ/アプリケーション起動、O は上位ディレクトリ移動などですが、表示モードによって、いろいろ変わります。
eff はキーバインドをカスタマイズできないのですか？
キーバインドをダイヤモンドカーソルベースにしたいのですが
1.20 より _eff に bindkey 命令でキーバインドを変更できます。ただし、１キーごとに変更しなければいけません。手軽にダイヤモンドカーソルベースのキーバインドにするには、添付の filenova.eff という設定ファイルを、_eff から include させてください。
include 文で、ファイルが存在しないというエラーが出ます。
include 文で相対パスを使うと、それは eff 起動時のカレントディレクトリが基準となります。ホームディレクトリを基準にするには、引数のファイル名の頭に「~/」を追加してください。 
	例：include "~/filenova.eff";
設定ファイル _eff を変更した際、すでに起動している eff で読み込む方法はないのでしょうか？
ファーラー画面で、設定ファイルにカーソルを移動してからF (Shift+F) をタイプしてください。
表示される背景色が違うファイルがあります。これ何ですか？
拡張属性 .LONGNAME を持ったファイルの場合、本来のファイル名ではなく、ワークプレースシェルで表示される名称が代わりに表示されます。背景色の違いはそれを示すものです。
