スクリプトのキャッシングにえらいバグがありました。 キャッシュにヒットした際に、 スクリプトの名前が直前に実行したコマンド名に化けてしまうのです。 たとえば、「test.pl↓」「less↓」「test.pl↓」と入力すると、 最後の test.pl は「/usr/local/bin/perl (フルパス)\test.pl」と 変換されなくてはいけないのが「/usr/local/bin/perl less」に 化けてしまうわけです。

これはスクリプト名をフルパスに変換するルーチンが、 キャッシュがヒットした際に呼び出されていなかったためでした。 つまり、フルパスが入っているバッファには前回実行した時の コマンドが入っていたというわけです。 しかも、直前のコマンドが内蔵コマンドだとハングしてしまうという... 1.35 からのバグでした。
